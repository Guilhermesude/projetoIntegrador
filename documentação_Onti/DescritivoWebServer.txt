 -- Descritivo de funcionamento de um gateway Web-Server -- 

 - Principais diferenças:

	* O tamanho da partição utilizada para servidor mariaDB na memoria RAM é configurada para ser maior;
	* As credênciais de acesso aos bancos de dados devem corresponder as mesmas exigidas pelo banco de dados:
		- Para aplicar a criptografia utilizada nas credenciais utilizar o software 
		  Criptografia_dos_arquivos.py contido neste diretorio.
	* O supervisor.py realizará a criação dos bancos de dados de forma dinamica funcionando tanto para 
	  web-server quanto para gateways comuns, se baseando na credencial de acesso do banco online,
	  quando o host de acesso for == 'localhost' então cria a versão web-server.
	* O arquivo montaBancos.sql contém comandos SQL para criar o banco de dados no servidor para 
	  posteriormente receber um dump dos bancos de dados.
	* Todo o site deve ser atualizado de acordo com a versão atual (ou com a versão designada para o gateway em questão)
          localmente.
	* O banco de dados reponsavel por simular um banco online contém rotina armazenada na procedure
	  controle_Historico, onde mantén um limit para a quantidade dos dados do banco.
	

 - Site:

	Após seguir o tutorial de como instalar e configurar o servidor apache todos os arquivos do site 
	contidos no diretorio html contido neste diretorio devem ser colocados na pasta www do raspberry
	conforme indicado no tutorial.

	- opções para melhorias:
	
	* Atualmente o site deve ser atualizado após se obter uma nova versão estavel do mesmo,
	  poderia ser adicionado um algoritmo para apontar para um servidor para realizar o download
	  dos arquivos e então atualizar somente um servidor de paginas web.
